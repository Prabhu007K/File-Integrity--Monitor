File Integrity Monitoring

Detect unauthorized file changes before they become a breach — baseline, hash, compare, alert.

01

What is this tool?

A File Integrity Monitor (FIM) watches a folder on your system, builds a trusted snapshot (baseline), and alerts you when files are created, modified, or deleted. It is used in security operations to spot tampering, malware drops, and config changes.

SHA-256 hashing Baseline compare One alert per change
02

How does it work?

File → SHA-256 → Baseline → Compare → Alert
1
Create baseline

Every file is hashed — fingerprints saved as “known good”.

2
Monitor

Re-hash on a timer and compare to baseline.

3
Alert once

Single timestamped alert per change — no spam.

04

Why SHA-256?

A hash is a fixed-length fingerprint of file content. Change one byte and the hash changes completely — so you detect tampering without storing full file copies.

baseline:  sample.txt → e3b0c44298fc1c149afbf4c8996fb924...
after edit: sample.txt → 9f86d081884c7d659a2feaa0c55ad015...
05

Alert types

CREATED

A new file appeared that was not in the baseline.

MODIFIED

Same path, different hash — content was altered.

DELETED

A file from the baseline is missing.

06

Real-world use

  • Detect malware or webshell drops in web directories
  • Watch critical config files (/etc, app settings)
  • Compliance requirements (PCI-DSS, SOC 2, NIST)
  • Post-incident forensics — know what changed and when
